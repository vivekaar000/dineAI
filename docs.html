Self-Hosting Setup Guide

Get Aura connected to the LLM running on your NAS in under 5 minutes.

What you'll need

A NAS or server on your local network capable of running an LLM backend (Ollama recommended). Most modern NAS units with Docker support will work.

1. Install Ollama on Your NAS

The easiest option is running Ollama via Docker on your NAS:

Docker
docker run -d --name ollama \
-p 11434:11434 \
-v ollama:/root/.ollama \
--restart unless-stopped \
ollama/ollama

If your NAS has an NVIDIA GPU, add --gpus all for hardware acceleration.

2. Pull a Model

Download a model to your NAS. Here are some good starting points:

Terminal
# Lightweight & fast (3B params)
docker exec -it ollama ollama pull llama3.2

# Powerful general purpose (8B params)
docker exec -it ollama ollama pull llama3.1

# Great for coding (7B params)
docker exec -it ollama ollama pull deepseek-coder-v2

# Compact & capable (3.8B params)
docker exec -it ollama ollama pull phi3

3. Connect Aura

Open Settings and configure:

1

API Endpoint — Set to your NAS IP, e.g. http://192.168.1.100:11434

2

API Format — Select "Ollama"

3

Test Connection — Click to verify, then select your model

That's it! Head to the chat and start talking to your AI.

Ollama

The recommended backend. REST API on port 11434. Works out of the box with Docker on most NAS systems (Synology, QNAP, TrueNAS, Unraid, etc).

Environment
# Allow connections from any host on your LAN
OLLAMA_HOST=0.0.0.0
OLLAMA_ORIGINS=*

LocalAI

OpenAI-compatible API. In Aura settings, select "OpenAI-compatible" format. Default port: 8080.

LM Studio

Start the local server in LM Studio (port 1234 by default). Select "OpenAI-compatible" format in Aura and set endpoint to http://your-ip:1234.

llama.cpp

Run the server with llama-server --host 0.0.0.0 --port 8080. Select "OpenAI-compatible" format in Aura.

CORS Issues

If Aura can't connect, your LLM server may be blocking cross-origin requests. For Ollama, set this environment variable:

Fix
OLLAMA_ORIGINS=*

For Docker, pass it as: -e OLLAMA_ORIGINS=*

Network Access

Firewall check

Make sure the port (e.g. 11434) is open on your NAS firewall and accessible from the machine running Aura.

FAQ

Can I use this from another device on my network?

Yes! Just serve the Aura HTML files from any web server on your network (even a simple Python HTTP server). All config is stored in your browser's localStorage.

Which models run well on a NAS?

It depends on your RAM. 8GB: Phi-3 (3.8B), Llama 3.2 (3B). 16GB: Llama 3.1 (8B), Mistral (7B). 32GB+: Mixtral (47B), Llama 3.1 (70B quantized).

Does any data leave my network?

No. Aura is a static HTML/JS app. Chat messages go directly from your browser to your LLM server. Nothing is sent to any external service.

Can I use a GPU on my NAS?

If your NAS has an NVIDIA GPU, add --gpus all to the Docker run command. AMD GPUs work with the ROCm version of Ollama.
